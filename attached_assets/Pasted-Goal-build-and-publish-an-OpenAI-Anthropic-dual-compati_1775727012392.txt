Goal: build and publish an OpenAI + Anthropic dual-compatible proxy API in a Replit pnpm monorepo.

【Environment Setup】
- Add OpenAI and Anthropic AI integrations in Replit Integrations through setupReplitAIIntegrations. This automatically injects AI_INTEGRATIONS_OPENAI_BASE_URL, AI_INTEGRATIONS_OPENAI_API_KEY, AI_INTEGRATIONS_ANTHROPIC_BASE_URL, and AI_INTEGRATIONS_ANTHROPIC_API_KEY. The user does not need to provide API keys, and usage costs are billed to Replit credits.
- Use requestEnvVar to request the Secret: PROXY_API_KEY (any string, used as the Bearer Token). If SESSION_SECRET already exists, skip it.

【API Server Changes】
File: artifacts/api-server/src/routes/proxy.ts

# Model list
- GET /v1/models:
  * validate the Bearer Token
  * return the model list:
    - OpenAI: gpt-5.2, gpt-5-mini, gpt-5-nano, o4-mini, o3
    - Anthropic: claude-opus-4-6, claude-sonnet-4-6, claude-haiku-4-5

# POST /v1/chat/completions (OpenAI-compatible interface)
- Validate Authorization: Bearer ${PROXY_API_KEY}. If validation fails, return 401.
- Route by model prefix:
  * models starting with gpt- or o -> OpenAI client
  * models starting with claude- -> Anthropic client
- Full Tool Call support:
  * pass through tools and tool_choice to OpenAI in the original format
  * for Anthropic, convert tool formats in both directions: function.parameters ↔ input_schema
  * map tool_choice for Anthropic: required -> any, specific function -> {type:"tool"}
- Message conversion:
  * convert role:"tool" into Anthropic tool_result content blocks
  * convert assistant tool_calls into tool_use blocks
  * parse JSON arguments into the input object
- Response conversion:
  * convert Anthropic tool_use blocks into OpenAI tool_calls
  * map stop_reason:"tool_use" -> finish_reason:"tool_calls"

# Streaming (stream=true)
- Set Content-Type: text/event-stream and X-Accel-Buffering: no.
- Call res.flushHeaders().
- OpenAI stream:
  * pass through chunks directly
  * call res.flush() on every chunk
- Anthropic stream:
  * convert content_block_start / delta / stop into OpenAI chunk format
  * convert streamed tool input_json_delta into function.arguments deltas
  * call res.flush() on every chunk
- Send a keepalive every 5 seconds with ": keepalive\n\n".
- On req.on("close"), clearInterval.
- Use try/finally to prevent 500 errors.

# Non-streaming (stream=false)
- OpenAI:
  * return the raw response directly
- Anthropic:
  * always use messages.stream().finalMessage() internally as a buffered path
  * Anthropic requires streaming internally for long requests; otherwise it returns 500 after 10 minutes
  * return OpenAI-format JSON

# Body size
- Set the Express body limit to 50mb.

# POST /v1/messages (native Anthropic Messages interface)
- Validate the Bearer Token in the same way.
- Accept the native Anthropic Messages API request format:
  * system
  * messages
  * tools
  * tool_choice
  * max_tokens
  * stream
  * and related fields
- For Claude models:
  * pass through directly to Anthropic
  * for non-streaming, internally use stream().finalMessage()
  * for streaming, forward all native SSE events:
    - message_start
    - content_block_start / delta / stop
    - ping
    - message_delta
    - message_stop
- For OpenAI models:
  * convert Anthropic messages into OpenAI messages:
    - tool_result -> role:"tool"
    - tool_use -> tool_calls
  * convert tool format:
    - input_schema -> parameters
    - map tool_choice accordingly
  * convert the OpenAI response into Anthropic Message format:
    - content[] blocks
    - stop_reason
    - usage.input_tokens / output_tokens
  * for streaming, synthesize a full Anthropic SSE event sequence, including tool call blocks:
    - content_block_start with type tool_use
    - input_json_delta
